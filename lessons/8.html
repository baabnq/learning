A Routine Signature is a comment that precedes the actual routine and specifies the routine's type. It is comprised of two parts: The type signature and the implementation signature.


Type Signature

The type signature usually comes first as part of the routine signature and details the in- and out-parameter names along with their types in calling order.
(InParam::InType, ...) => (OutParam::OutType, ...)
Parameters are delimited by commas, and the list of them is enclosed in parenthesis. In the parameter, name comes first, then two colons, then the type.

Implementation Signature

The implementation signature follows after the type signature. It usually consists of either a textual description of the function and working of the routine or a parallel implementation of the routine in a tacit programming language like Haskell or Uiua.
