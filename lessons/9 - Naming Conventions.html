Note that: These conventions are only guidelines and not rigid rules. It's generally a good idea to follow these, but you can to whatever you want, i (probably) won't stop you.

Routines and Sublabels

Labels in routines always follow this format:
lab <module name>::<routine name>::<sublabel 0>:: ... ;
This is known as identifier scoping and done because labels, much like variables, have no inherent scope and are globally accessable.

Variables

Variables often use a form of Hungarian Notation to indicate their type. Variables are prefixed with a shorthand identifier, the most common of which are:

Prefix Meaning Example
"s" string

new 'hello world' _sHello;

"p" pointer

static 100 _pBuffer;

"i" index

put _iLoop = 0;

"fp" floating-point value

put _fpValue = _Floats::Zero;

While variables are also all scopeless, they are almost always only used locally. As such, they usually don't use identifier scoping. The arising problem of variable collision between different routines and modules is solved through variable insulation. When a variable is, however, is intended for global use, it also uses identifier scoping. See the example above _Floats::Zero, where this is used for a globally accessable constant, stemming from the floating-point module.
